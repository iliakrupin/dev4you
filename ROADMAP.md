# Roadmap

Чек-лист доработок поверх MVP. Порядок — приоритет (от ближайших к дальним).

## Сделано

- [x] **Мьютекс на main (атомарный)** — одна активная задача за раз, подкреплено частичным уникальным индексом `one_active_task` в БД + SELECT-проверка с дружелюбным 429. Требует `pnpm db:push`.
- [x] **Watchdog зависших задач** — `/api/cron/watchdog` (каждые 5 мин) добивает задачи, застрявшие в активном статусе >5 мин, чтобы они не держали мьютекс вечно.
- [x] **Подписи webhook'ов + fail-closed** — `/api/webhooks/{github,vercel}` проверяют HMAC; reset переведён на POST-only + опциональный токен; initData fail-closed + replay-защита. Единая точка мержа (`finalizeImplement`).

## Ближайшее

- [ ] **Очередь с местом в строю** — мьютекс уже сериализует (новая задача → 429, если есть активная). Дальше: показывать зрителю «вы №N в очереди» вместо отказа.
- [ ] **Auto-revert main при production failure** — сейчас webhook лишь помечает задачу `failed`; main остаётся со сломанным коммитом до ручного reset. Автоматически реверти́ть последний коммит задачи.
- [ ] **Расширить TaskCard** — перенести timeline, план, ошибку, ссылки на PR/preview прямо на карточку, чтобы детальный экран `/tasks/[id]` был не нужен (он уже редиректит на главную).
- [ ] **Тёмная/светлая тема Telegram** — `Telegram.WebApp.colorScheme` менять CSS-переменные на лету.
- [ ] **Регистрация Mini App в BotFather** — подключить `dev4you_bot` Menu Button → ссылка на `https://dev4you-pi.vercel.app`.
- [ ] **Анти-abuse** — простой rate limit на `telegram_id` (1 задача / 10 мин), фильтр откровенно вредных постановок.
- [ ] **Live-лог LLM-токенов в UI** — видеть как агент «думает» в реальном времени (через SSE из task_events). Сильнее WOW для технарей.
- [ ] **Privacy policy** — обязательна для публикации Mini App в каталоге Telegram.

## Среднесрочное

- [ ] **Очередь с прогрессом для зрителей** — «вы №3 в очереди, ETA 6 минут».
- [ ] **Visual regression через Playwright** — после preview build делать скриншот, LLM сравнивает с целью задачи (semantic verification вместо текстовой).
- [ ] **Несколько LLM-моделей** — выбор модели на агентах analysis/implement отдельно (Claude / GPT / Qwen / DeepSeek).
- [ ] **Несколько проектов в одном workspace** — переключатель «над каким репозиторием работаем».
- [ ] **Несколько workspace** — изоляция групп проектов и команд.
- [ ] **Голосовая постановка** — Whisper → текст задачи.
- [ ] **Интеграция с Redmine / Jira / Linear** — задача приходит из внешнего трекера, статус обновляется обратно.
- [ ] **pgvector + RAG по коду** — для задач, где агенту нужно находить релевантные файлы из большого репо (не только whitelist).

## Долгосрочное

- [ ] **Hierarchical multi-agent** — сейчас один агент с 4 этапами. Реальные 4 субагента (analysis / implement / test / deploy) с эскалацией к оркестратору при провале.
- [ ] **Self-improving prompts** — на провалах накапливаем антипримеры, дообучаем prompts через few-shot.
- [ ] **Multi-tenant SaaS** — раздельные команды, биллинг, изолированные БД.
